## 1. NCBI Resources & Retrieving Data

The National Center for Biotechnology Information (NCBI) is the primary hub for accessing sequence data.

* **Accessing NCBI:** Visit the [NCBI website](https://www.ncbi.nlm.nih.gov/). Use the top search bar and dropdown menu to navigate different databases.
* **Key Databases:**
* **Organelles:** Contains complete genome sequences of organelles like mitochondria and chloroplasts.
* **Taxonomy:** Provides classification and phylogenetic lineages for over 160,000 organisms.
* **Nucleotide (DNA & RNA):** Includes GenBank and RefSeq for storing sequence data.
* **Genomes & Maps:** Information on overall genome structure and metadata.


* **Advanced Search Operators:**
* `AND`: Finds results containing all keywords (e.g., `"mitochondrial AND frog"`).
* `OR`: Finds results containing any keywords (e.g., `"mito OR mitochondrial"`).
* `NOT`: Excludes keywords (e.g., `"Hymenoptera AND mitochondrial NOT Nasonia"`).
* `*` (Wildcard): Replaces parts of a word (e.g., `"mito*"`).


* **Downloading Data:** Locate your sequence, click **"Send to" > "File" > "FASTA"** (or GenBank format) **> "Create File"** to download it to your local machine.

## 2. Alignment Examples & Molecular Markers

When building phylogenies, you will work with different types of molecular markers, each with unique evolutionary constraints. Understanding your Operational Taxonomic Units (OTUs) is crucial.

* **Ribosomal RNA (e.g., 16S rRNA in Termites):** A non-coding gene widely used in phylogenetics. Because it is an RNA structural gene, it does not translate into proteins, meaning it has no codon structure.
* **Protein-Coding Mitochondrial Genes (e.g., CytB in *Goniurosaurus*):** Encodes the cytochrome b protein. These sequences are read in triplet codons and require specific evolutionary models based on codon positions.
* **Protein Sets (e.g., Aculeata Hymenoptera or Spider Silk Genes):** Amino acid alignments from ortholog groups. Note that the sequence length in amino acids will differ from the original nucleotide length.

## 3. Viewing, Aligning, and Editing with AliView

[AliView](http://www.ormbunkar.se/aliview/) is a lightweight, fast viewer and editor for multiple sequence alignments.

* **Supported Formats:** FASTA, Nexus, Phylip, Clustal, and MSF.
* **Automatic Alignment:** Select all sequences (`Ctrl+A` or `Cmd+A`), then navigate to **Align > Align with MUSCLE or MAFFT** to generate a multiple sequence alignment.
* **Translation:** You can dynamically translate nucleotide sequences to amino acids via **Tools > Translate Nucleotides**, which is highly useful for checking reading frames in protein-coding genes.
* **Saving and Exporting:** Once aligned, use **File > Save As** to export your polished dataset. For coding regions, you can use `File > Save as codonpos Nexus` to automatically cluster First, Second, and Third codon positions into a partitioned file.

## 4. Concatenation & Codon Partitioning

Instead of relying on command-line scripts to merge different genes, modern graphical interfaces handle this automatically.

### Why Partitioning Matters

When working with protein-coding DNA, sequences are read in triplets (codons). Each position experiences different evolutionary pressures:

* **1st Position:** Moderate selective constraints; changes often result in non-synonymous amino acid alterations.
* **2nd Position:** Highly conserved; changes almost always alter the amino acid.
* **3rd Position:** Lower selective constraints; changes are often synonymous due to genetic code redundancy, making them prone to rapid saturation.
* *Note: Codon partitioning should **never** be applied to non-coding ribosomal genes (like 16S) because they are not translated.*

### Automated Concatenation

When using multiple markers (e.g., combining `COX1`, `CYTB`, and `ND4` in vertebrates), you must concatenate the aligned sequences. **raxmlGUI 2.0** features an automated concatenation and partitioning tool. By loading multiple alignment files into the software sequentially, it automatically parses taxa names, combines matching sequences, and specifies the correct partition settings for you.

## 5. Phylogenetic Inference with raxmlGUI

**raxmlGUI 2.0** is a user-friendly graphical front-end for RAxML and RAxML-NG, which are among the most popular and efficient software tools for inferring phylogenetic trees using Maximum Likelihood.

* **Installation:** Download the latest 2.0 version from the official repository at [antonellilab.github.io/raxmlGUI/](https://antonellilab.github.io/raxmlGUI/).
Here is the step-by-step tutorial section to add to your document, detailing how to perform a complete Maximum Likelihood analysis with concatenation, model testing, and bootstrapping using raxmlGUI 2.0.

### Step 1: Loading and Automatically Concatenating Alignments

1. Open raxmlGUI 2.0 and click the **Load Alignment** button to open your first `.fasta` (or `.phy` / `.nex`) alignment file.
2. Instead of using command-line tools like Phyutility to merge the rest of your genes, simply click **ADD ALIGNMENT** in the top-right section of the interface and select your second gene. Repeat this for any additional markers.
3. **What happens under the hood:** raxmlGUI 2.0 automatically parses the taxa names, drops empty sequences (or adds missing data where necessary), concatenates everything into a single combined alignment file, and writes a partition file specifying the exact start and end positions for each gene.

### Step 2: Model Testing with ModelTest-NG

Since different partitions (genes or codon positions) evolve under different constraints, they need different evolutionary substitution models.

1. In the **INPUT** panel (green section), look for the *Substitution model* area.
2. Click the **OPTIMIZE** button.
3. **What happens under the hood:** raxmlGUI seamlessly runs **ModelTest-NG** in the background, testing hundreds of evolutionary models independently for each of your partitions. It will automatically apply the best-fitting model (typically using the Bayesian Information Criterion, BIC) to each specific partition before the main run.

### Step 3: Setting up the Analysis and Bootstrapping

Now we must configure the actual tree search.

1. Go to the **ANALYSIS** panel (yellow section).
2. From the *Analysis* dropdown, select **ML + rapid bootstrap** (or **ML + thorough bootstrap**). This tells the software to search for the best ML tree and calculate branch support in one continuous pipeline.
3. Under *Reps* (Replicates), change the value to **1000** to perform 1000 bootstrap pseudo-replicates.
4. Select your **Outgroup** from the automatically generated dropdown list.
5. Click **RUN** on the right side of the screen.

### Step 4: Understanding the Output and RAxML Commands

While raxmlGUI 2.0 provides a friendly interface, it is essentially building a command-line prompt and sending it to the RAxML engine. You can view this exact command in the **CONSOLE** window.

For the pipeline we just built, the background command will look something like this:
`raxmlHPC -f a -m GTRGAMMA -s concatenated_alignment.fasta -q partition_file.txt -p 12345 -x 12345 -# 1000 -n my_analysis_output`

**Breakdown of the RAxML commands:**

* `-f a`: Tells RAxML to conduct a rapid Bootstrap analysis and search for the best-scoring ML tree in one single program run.
* `-s`: Specifies the sequence alignment input file.
* `-q`: Specifies the partition file that assigns different models of substitution to different regions of the alignment.
* `-m`: Specifies the substitution model. Even if ModelTest-NG assigned different models to different genes, this flag is required to specify the overall model of *rate heterogeneity* (e.g., `GTRGAMMA` or `GTRCAT`).
* `-p`: A random number seed for the parsimony starting tree (required to make analyses reproducible).
* `-x`: A random number seed for the rapid bootstrapping algorithm.
* `-#` (or `-N`): Specifies the number of bootstrap replicates (in our case, 1000).
* `-n`: Specifies the prefix name for all the output files generated by this run.



* **Interpreting Outputs:** Once the run is complete, raxmlGUI provides a list of output files. The most critical file is typically named `RAxML_bipartitions.input.tre`, which contains your final Maximum Likelihood tree with bootstrap support values mapped directly onto the nodes.

## 6. Tree Visualization

After generating your `.tre` file, you will need to visualize and customize the tree.

* **[FigTree](http://tree.bio.ed.ac.uk/software/figtree/):** A powerful local application to view, edit, and export your phylogenetic trees. Be sure to enable node labels to display your bootstrap support values.
* **Online Alternatives:** * [Phylo.io](https://beta.phylo.io/viewer/)
* [TreeViewer](https://treeviewer.org/)
